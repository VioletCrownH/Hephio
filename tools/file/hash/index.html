Frequently Asked Questions

What is a file hash used for?

A hash is a unique cryptographic fingerprint of a file's contents. If anything in the file changes — even a single bit — the hash changes completely. This makes hashes the standard method for verifying that a file hasn't been corrupted or tampered with.

Which hash algorithm should I use?

SHA-256 is the recommended choice for most purposes today. MD5 and SHA-1 are still widely used for basic integrity checks but are no longer considered cryptographically secure against intentional tampering.

Can I verify that two files are identical?

Yes. Hash both files — if they produce the same hash value, the files are byte-for-byte identical. This is faster and more reliable than comparing files manually.
